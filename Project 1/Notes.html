Part 1: Thinking Like a Programer

5 Thinking Methods

It's important to remember that when you are in teh planning stages of writing your code, that you need to think like a computer. Since computers are "so smart", they will literally do exactly as you tell them to. This makes them very unforgiving since a small mistake by a programmer can cause huge problems in a program.

Here are 5 methods to consider:
  1. 1. Procedural Thinking
  2. 2. Abstract Thinking
  3. 3. Systems Thinking
  4. 4. Technological Empathy
  5. 5. Debugging
What is a Web Page

A web page is a text document written in a language called HTML. Web browsers read these documents, and then interpret them to display text, images, graphics and videos as defined in your .css file. We will go through CSS and what that does later.

How Coding Works

For our purposes when refering to a programmer, we will be talking about HTML coding. A programmers write text or a set of instructions, in a language that a computer can understand. HTML Tags are the hidden keywords or values within a web page that define how the browser must format and display the content. Most tags must have two parts, an opening "< >" and closing "</ >".

Forgetting Something?

It's okay to forget! Often I found myself googling anything and everythign related to HTML coding (and still do). If you're like me and not someone who programs all the time, you'll find yourself using google and other useful sites to figure out how to code something correctly.

Part 2: Adding CSS to your HTML and Organizing Coding

What is CSS?
CSS or Cascading Style Sheet is code written to control the "style" of HTML elements. CSS helps define font, size, color, spacing, border and location of HTML information on a web page, and can also be used to create a continuous look throughout multiple pages of a website.
  1. 1. Write CSS in the head of your HTML - Good for small projects as you will be able to write the the CSS above the text that you want to display.
  2. 2. Link your HTML to a separate CSS file - Good for larger projects as you will be able to create tags in a separate .css file which your HTML will call. This is used when many elements of your code are similar. This helps in avoiding repetition of html tags, keeps your code neat, organized and saves time when you're formatting multiple lines in the same or similar text style, font size, alignment etc.
  3. 3. Write your style inline with your HTML - Probabaly the worst way to give style to your HTML as this will create a lot of repeated code. If you find yourself doing this, you may want to take a step back and think about your approach to the entire project.
